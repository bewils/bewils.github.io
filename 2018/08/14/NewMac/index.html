新 Mac 配置

2018-08-14

前段时间手欠更新了系统, 体会到了 mac 变砖的快感. 出问题的情况和 https://www.loli.ren/archives/43/?from=groupmessage&isappinstalled=0 这篇解决方案描述的大体一致. 然而用这个解决方案并没有办法, 最后还是最暴力用外接 U 盘装了个系统强行开机, 把东西做好备份后格了整个硬盘. 过去既没钱买 time machine 也觉得 5G 的 iCloud 够用了, 这次真是死得惨. 鉴于目前还是穷就乖乖先买着便宜的 iCloud, 把怕丢的东西全扔上去. 得到了一台崭新的 mac 后又到了人生最喜忧参半的配环境时刻了. 这次就记录下我的一些习惯的配置, 主要是各种工具和软件的安装, 顺便做个安利.

iTerm2

mac 自带的终端不是很喜欢, 功能也相对少, 先换个终端工具 https://www.iterm2.com

zsh

zsh 提供了比 bash 更全面的功能, 至于 shell 语法上的不同如果不是专门写 shell 不需要关心(更何况写 shell 都在第一行加了 #!/bin/bash 写的本来就是 bash shell) https://github.com/robbyrussell/oh-my-zsh/

zsh 中有很多插件, 可以按照官网的介绍来弄, 虽然我平时只会用 git

zsh 默认的主题不是很清晰美观, 推荐使用 ys 这个主题

Homebrew

万物之源 Homebrew, 整个 mac 系统上所有命令行工具的管理器. 所有库都用 homebrew 的好处就是装卸都有保障 https://brew.sh

Python

有了 Homebrew 之后就可以开始安装各种方便的工具和编程语言了, 安装 python 的时候需要注意一些问题:

首先 brew install python 其实是安装的 python3, 而 brew install python2 才是安装的 python2.

其次, 因为 macOS 是自带 python2 的, 在 /usr/bin 目录下, 其实直接使用也不要紧, 只是为了统一管理我自己还是重新装了个 python2. 这样系统中就有了两个 python2 的版本, 一个是系统使用的在 /usr/bin, 另一个是用户使用的在 /usr/local/bin, 使用时通过 python 和 python2 命令来区分.

如果自己安装了 python2 的话需要通过 which python 来确认 python 命令是否还是指向了 /usr/bin 路径下的那个 python, 如果不是的话应该是 PATH 出了问题, 可以通过修改 .bashrc/.zshrc 或者最暴力修改系统的 /etc/paths 来解决

比如我的 /etc/paths 就是这样的, 这样在执行 python 的时候就会找到 homebrew 安装的版本, 暴力解决方案就 sodu vim 再重启就行了(

$ cat /etc/paths
/usr/local/bin
/usr/bin
/bin
/usr/sbin
/sbin

Node

因为是用 hexo 来写博客, 电脑还是需要一个 node 环境, 直接装就行, 对于 node 开发的话可以用 nvm 来管理 node 版本, 不过我还是倾向简单的 n, 毕竟 n 用 npm 就能装了

vimplus

用 vim 来写写小脚本还是挺方便的, vim 又很难用, 直接装个 vimplus, 这个会在 vim 中安装大量插件也会用 homebrew 安装许多依赖. 但是因为 vim 毕竟还是很轻量, 在打开大文本文件的时候就算带插件还是比 vscode/atom 之类的要快不少. https://github.com/chxuan/vimplus

annie

命令行视频下载工具, 吹爆, 支持各大视频网站的下载, 可以通过 homebrew 直接安装

一个无敌的功能是可以直接下载播放列表中的所有视频, 甚至还对 b 站做了特别的优化, 只要输入 av 号即可下载不用输出整个 url https://github.com/iawia002/annie

Alfred

直接把系统的 spotlight 关掉换成 Alfred 就行, 除了能够方便启动各种程序和打开文件外, Alfred 还有大量的 workflow 可用.

一个 workflow 集合 https://github.com/zenorocha/alfred-workflows, 基本够用

另外还有一个查字典的 https://github.com/liberize/alfred-dict-workflow, 下载后推荐去配置里把默认字典即 config.json 中的 ‘default’ 字段从 ‘nj’(牛津, 系统默认) 改成 ‘yd’(有道), 系统的字典什么都查不出来

Go2Shell

下载后打开可以设置默认打开的 shell 为 iterm2, 并且在 Finder 中安装后可以看到文件夹中多了一个命令行的图片, 点一下就可以在当前路径打开一个 shell, 再也不用为路径发愁了

iina

纯 Swift 写的视频播放器, 非常轻巧, 简洁现代就像 QuickTime 一样干净(当然功能不知道强到哪去了) https://github.com/lhc70000/iina

除了用键盘控制进度音量外还支持手势, 用触摸板或者 magic mouse 非常舒服.

自动识别字幕, 播放速度调整, 支持画中画

我非常喜欢的一个功能是一个文件夹中的系列视频 iina 能够自动识别成播放列表, 并且正确排序, 看起剧来非常爽(vlc 的问题就是用字符串排序, 第1集后面是第10集而不是第2集)

The Unarchiver

mac 解压 rar, 没广告, 免费, Appstore 下载

Magnet

是用快捷键来管理窗口, 直接将窗口调整到全屏幕, 半屏幕, 1/3, 2/3 等比例

Typora

非常好用的 markdown 书写工具, 比较纯粹, 就是单纯的开文件编写, 没有什么分类什么同步之类的功能 https://www.typora.io

特色之一是不区分编写和预览, 所见即所得

另一个就是有打字机模式和专注模式, 非常有趣, 推荐体验

还有就是可以换主题, 默认的 github 主题非常眼熟了, 最近去看甚至还有了 vue 官方文档的主题, 因为 Typora 的主题通过 css 控制, 因此想自定义也非常方便

VSCode

对 TS, Python 等支持很好, 可以找到语言环境的路径, 能够轻松查看函数的调用情况

很强的自定义能力, 并且内部直接继承了 shell, 一边编写一边运行毫无压力 https://code.visualstudio.com

VirtualBox

虚拟机环境, 比较简陋但和 Parallels Desktop 的优势是免费, 而且像我跑 ubuntu server 版也不需要什么高级功能

https://www.virtualbox.org

Xcode

如果做苹果相关开发就装全部, 否则只需要安装 Command Line Tools 即可. 不过 CLT 也不急着装, 因为当你执行命令需要的时候系统会弹窗提示, 点下安装就行了

Source code pro

Adobe 出品的为了编程而生的等宽字体, 看起来非常舒服, 直接安装在系统的字体集中就可以在各个编程软件(iterm2, VSCode, Xcode)中修改字体了 https://github.com/adobe-fonts/source-code-pro

Nord

一套基本支持全部代码编辑器的配色方案, 深色柔和系看起来比较舒心, 所以我将 VSCode 和 iterm2 都改成了 Nord 配色. 前面安装过的 vimplus 会将 vim 的配色默认改成 One Dark, 我觉得这样就挺好, 可以立刻区分出来哪个 shell 正在进行 vim 编辑, 哪些是普通的 shell

目前使用中就只用到这些比较方便的工具和软件了, 接下来会持续更新, 找到什么新的更好的就分享出来. 虽然是新 mac 的配置方案, 以防变砖和换电脑, 不过还是希望自己这两年内都不要用到这篇文章吧(
